# 项目部署总结说明

## 1. 使用了哪些 AI 工具

| 工具 | 用途 |
|------|------|
| **Codex** | 编写/修改项目代码、配置部署文件、调试 404 错误、引导 Git 操作与 Render 部署流程 |
| **DeepSeek (deepseek-v4-pro)** | 作为 Codex 底层的代码生成与推理模型,完成代码修复、路径推导、配置生成等任务 |

## 2. AI 帮助完成了哪些工作

- 诊断 Vercel 部署 404 问题,识别出 SPA 路由未配置,创建 `vercel.json` 添加 rewrites 规则
- 修复 `vite.config.js` 中硬编码的绝对路径（含中文乱码）导致的沙箱构建失败
- 创建 `render.yaml` 部署配置文件,配置 Render Web Service 的构建命令、启动命令、环境变量
- 多次修复 `render.yaml`: 添加 `rootDir` 解决路径查找问题,调整命令适配 Render 目录结构
- 升级 `requirements.txt` 依赖版本（pydantic 2.5.2 → 2.13.4 等）,解决 Python 3.14 + pydantic-core 编译（Rust/maturin）失败问题
- 重写 `main.py`,修复种子数据中文乱码（`鍐瀹?` → `冯家宝`）,添加 dist 路径调试日志
- 发现并修复 `.gitignore` 中 `dist/` 规则导致前端构建产物被排除、Git 未上传的根本问题
- 引导配置 Git SSH Key、添加公钥到 GitHub、设置 `GIT_SSH_COMMAND` 环境变量解决中文路径编码导致的 SSH 失败

## 3. 自己手动修改了哪些内容

- 在 GitHub 创建 `eat-less/personal-website` 仓库
- 在 Render 注册账号、创建 Web Service、手动填写 Root Directory / Build Command / Start Command
- 生成 SSH Key（`ssh-keygen -t ed25519`）
- 将公钥添加到 GitHub Settings → SSH Keys
- 手动在 Render Dashboard 点击 Manual Deploy → Clear build cache & deploy
- 多次执行 `git add / commit / push` 将本地修改推送到 GitHub
- 在终端执行 Codex 提供的命令（`git push`、设置环境变量 `$env:GIT_SSH_COMMAND` 等）

## 4. 遇到的问题及如何通过AI解决

| 问题 | AI 如何解决 |
|------|------------|
| Vercel 部署后访问 404 | 分析出缺少 SPA 路由配置,创建 `vercel.json` 添加 `rewrites` 规则将所有路径重写到 `index.html` |
| `npm run build` 失败,esbuild 报 `Cannot read directory` 和路径乱码 | 定位到 `vite.config.js` 中 `fs.allow` 硬编码了含中文乱码的绝对路径,移除了该配置 |
| Render 构建报 `ERROR: Could not open requirements file: backend/requirements.txt` | 分析出 Render 工作目录与仓库结构不匹配,添加 `rootDir` 配置修复路径 |
| Render 构建报 `pydantic-core` 需要 Rust/maturin 编译失败 | 升级 `requirements.txt` 依赖到兼容 Python 3.14 的新版本,使用预编译 wheel 绕过 Rust 编译 |
| 部署成功后访问仍 404,FastAPI 没有挂载前端静态文件 | 发现 `.gitignore` 中 `dist/` 规则导致前端构建产物未被 Git 上传,移除该规则并重新推送 |
| `git push` 报 `Failed to connect to github.com` | 引导使用 SSH 替代 HTTPS,配置 SSH Key |
| Git 内置 SSH 因中文用户名路径编码异常（`\267\353\274\322\261\246`）无法读取 `.ssh/` 目录 | 设置 `GIT_SSH_COMMAND` 环境变量指定系统 SSH + 私钥路径 + known_hosts 路径,绕过 Git 内部 SSH |
| SSH 密钥路径含反斜杠被转义导致 `Identity file not accessible` | 将路径中的反斜杠 `\` 改为正斜杠 `/` |
