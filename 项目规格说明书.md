## 1. 项目概述

为抖音**个人号**（日常分享+自拍类博主）搭建自动回复智能体，覆盖私信和评论，回复风格积极正能量、有边界感。

## 2. 核心约束（已确认）

| 约束项 | 决定 |
|--------|------|
| 账号类型 | 个人号（无官方 API，必须浏览器自动化） |
| 运行环境 | 本地电脑（开机才跑） |
| 内容领域 | 日常生活 + 自拍 |
| 消息量 | 50~200 条/天 |
| 边界模糊时 | 推到待审核队列，人工决定 |
| 评论范围 | 仅回复自己视频下的评论 |

## 3. 技术栈

```
浏览器自动化: DrissionPage (首选，中文文档好，无需 WebDriver)
AI 回复引擎:  DeepSeek API (deepseek-chat)
后端框架:     FastAPI + uvicorn
数据库:       SQLite → 后续可迁移 MySQL
任务调度:     APScheduler
配置管理:     pydantic-settings + .env
前端(二期):   Vue 3 + Element Plus
```

## 9. 各模块实现规格

### 9.1 浏览器客户端

`agent/browser/client.py`：

- 用 `DrissionPage.ChromiumPage` 启动浏览器
- 加载 `cookies.json` 恢复登录态
- 提供 `goto_messages()`（跳到私信页）和 `goto_comment_management()`（跳到创作者服务中心评论管理）方法
- 设置反检测参数：`--disable-blink-features=AutomationControlled`
- 初始化时检查页面 title 是否包含"抖音"来验证登录状态

### 9.2 私信监控器

`agent/monitor/private_msg.py`：

- 继承 `base.py` 的 `BaseMonitor` 抽象类
- APScheduler 定时任务（间隔：`PRIVATE_MSG_INTERVAL_SECONDS`）
- 逻辑流程：
  1. `page.get(''https://www.douyin.com/messages'')`
  2. 等待会话列表加载（DOM 元素因抖音更新会变，需要灵活选择器）
  3. 遍历会话项，提取：`(会话ID, 对方昵称, 最后一条消息内容, 时间戳)`
  4. 对比数据库 `replied_messages` 表去重
  5. 新消息 → 传给 `filter.boundary.check()`
  6. 通过过滤 → 传给 `reply.generator.generate()`
  7. 生成回复 → 传给 `reply.scheduler.enqueue()`

### 9.3 评论监控器

`agent/monitor/comment.py`：

- 定时任务（间隔：`COMMENT_INTERVAL_SECONDS`）
- 跳转创作者服务中心：`https://creator.douyin.com/comment-manage`
- 只提取自己视频下的新评论（按时间倒序）
- 去重 + 过滤 + 生成 + 调度流程同私信

### 9.4 边界过滤器

`agent/filter/boundary.py`：

**第一层：关键词/规则过滤（本地，零成本）**

拦截以下类型：
- 广告引流：含微信/QQ/手机号/链接
- 辱骂/人身攻击：脏话词库
- 政治敏感：敏感词词库
- 暧昧/搭讪："约吗"、"加个微信"、"小姐姐哪里人"、"单身吗"
- 索要隐私：问全名、地址、收入等
- 商业合作（模糊的）：消息中含"合作"但不明确的 → 推到待审核队列

**第二层：AI 语义判断（调用 DeepSeek，成本可控）**

当第一层无法确定时，调用 LLM 二次判断，要求输出 JSON：

```python
# System prompt 设计
"""
你是一个内容安全审核助手。判断以下抖音用户发给博主（日常生活博主，分享生活与自拍）的消息是否可以安全回复。

回复规则：
- 正常互动（夸赞、提问、讨论内容）→ 安全
- 搭讪、暧昧、约会暗示 → 不安全
- 广告、引流、诈骗 → 不安全
- 辱骂、引战、负能量 → 不安全
- 模糊不清或踩线的内容 → 不确定

请只回复JSON格式：
{"safe": true/false, "reason": "一句话原因", "uncertain": true/false}
如果 "uncertain": true，这条消息会被推到人工审核队列。
"""

# User prompt
"""消息内容：{message_content}"""
```

**输出规则：**
- `{"safe": true}` → 进入回复生成
- `{"safe": false}` → 静默丢弃 + 记录日志
- `{"uncertain": true}` → 写入 `pending_review` 表

### 9.5 AI 回复生成器

`agent/reply/generator.py`：

**System Prompt 设计：**

```
你是一个抖音博主的AI助手，帮忙回复粉丝的私信和评论。

博主身份：{owner_name}，{owner_bio}

回复要求：
1. 积极正能量，传递善意和鼓励
2. 保持适当的距离感和边界感：
   - 不透露个人隐私（全名、地址、联系方式等）
   - 不答应私下见面或约会请求
   - 不参与暧昧对话
   - 对方搭讪时礼貌但疏离地转移话题
3. 遇到广告、骚扰、引战消息统一回复"谢谢关注"
4. 回复简洁（不超过100字），语气亲切但保持社交距离
5. 不讲政治、不传谣、不发表争议性观点
6. 不主动索要对方联系方式

对话历史（最近3轮）：{context}
粉丝最新消息：{message}

请生成合适的回复（只输出回复内容，不要前缀）：
```

**API 调用：**
- 模型：`deepseek-chat`
- temperature：0.7
- max_tokens：200
- 最近 3 轮对话历史作为 context 传入

### 9.6 回复调度器

`agent/reply/scheduler.py`：

**调度逻辑：**

```
[待回复消息]
  → 检查今日总回复量 < MAX_REPLIES_PER_DAY？
    → 否 → 丢弃 + 记录日志
  → 检查最近1分钟回复量 < MAX_REPLIES_PER_MINUTE？
    → 否 → 等待直到窗口放行
  → 检查该会话在 DEBOUNCE_MINUTES 内是否已回复？
    → 是 → 跳过（去重）
  → 加入执行队列
  → 执行回复
  → 写入 replied_messages + reply_log
```

**实现方式：**
- 内存队列 (`collections.deque`) + 锁 (`threading.Lock`)
- 一个后台线程消费队列
- 每次执行前检查 `reply_log` 表的最近记录做限频

### 9.7 回复执行器

`agent/reply/executor.py`：

```python
# 伪代码核心逻辑
def execute_private_reply(page, conversation_selector, text):
    """
    page: DrissionPage 的 ChromiumPage
    conversation_selector: 当前会话的 DOM 元素
    text: 要回复的内容
    """
    # 1. 点击进入该会话（如果还没进入）
    # 2. 等待输入框加载
    input_box = page.ele(''css:div[contenteditable="true"]'') or page.ele(''tag:textarea'')
    # 3. 清空 + 填入文字
    input_box.clear()
    input_box.input(text)
    # 4. 模拟人类打字延迟（0.5~2 秒随机）
    import time, random
    time.sleep(random.uniform(0.5, 2.0))
    # 5. 点击发送按钮
    send_btn = page.ele(''text:发送'') or page.ele(''css:[data-e2e="send-btn"]'')
    send_btn.click()
    # 6. 记录日志
```

### 9.9 主入口

`agent/main.py`：

```python
def main():
    # 1. 初始化数据库
    from agent.storage.database import init_db
    init_db()

    # 2. 初始化浏览器
    from agent.browser.client import DouyinBrowser
    browser = DouyinBrowser()

    # 3. 启动 APScheduler 调度器
    from apscheduler.schedulers.background import BackgroundScheduler
    scheduler = BackgroundScheduler()

    from agent.monitor.private_msg import PrivateMsgMonitor
    from agent.monitor.comment import CommentMonitor

    scheduler.add_job(
        PrivateMsgMonitor(browser).run,
        ''interval'',
        seconds=settings.private_msg_interval_seconds,
        id=''private_msg''
    )
    scheduler.add_job(
        CommentMonitor(browser).run,
        ''interval'',
        seconds=settings.comment_interval_seconds,
        id=''comment''
    )
    scheduler.start()

    # 4. 启动 Web 服务
    import uvicorn
    uvicorn.run("agent.web.app:app", host="0.0.0.0", port=8000)

if __name__ == "__main__":
    main()
```

## 11. 风险提示

| 风险 | 应对 |
|------|------|
| 抖音 DOM 变化导致选择器失效 | 选择器配置化，异常时写日志 + 告警 |
| 封号 | 限频（每分钟≤5条）、模拟人类打字延迟、不在深夜跑 |
| AI 回复不合规 | 边界过滤器做双保险（关键词 + AI 判断） |
| Cookie 过期 | 每次启动检查登录态，失效则提示手动扫码 |
